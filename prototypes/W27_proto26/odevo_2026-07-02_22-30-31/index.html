Our read

Odevo is moving from platform growth to production AI across many local companies. The AI role is not a demo role. It names case triage, document intelligence, invoice insight, and OdevoGPT. Jason's IT operations lens is useful here because adoption depends on stable systems, not only good models.

Where we'd start

Mobile field flow

Upkept Assets release notes call out dispatch, offline work, cleaner work-order cards, and association selection. We would map those flows and add regression tests before wider rollout.

AI trust path

Public HOA discussion is skeptical of broad AI claims. Start with narrow case triage, clear review steps, and logs that show why each route was picked.

What we'd do

Use a dedicated squad under Odevo's Product Owner and engineering leads. Keep scope tight for a six-week pilot, then extend the same team if it works.
