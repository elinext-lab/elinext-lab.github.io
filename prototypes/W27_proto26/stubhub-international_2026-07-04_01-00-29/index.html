Scaling ticket trust while the backend team grows

We saw StubHub International hiring a backend engineer for the distributed platform that powers the marketplace. That signal matches a bigger moment: ticket transfer trust, seller flows, and mobile performance need steady delivery.

Our read

The backend role says the marketplace platform is carrying more product load, not just more tickets. StubHub's own job post names Java or Kotlin microservices, Kafka, and REST or async APIs for a high-throughput marketplace. Recent app reviews point to the hard edge of that system: ticket handoffs, seller proof, refunds, and pop-ups can break trust fast. Bob has also said AI should solve real ticket anxiety, not add noise. If these flows stay split, support volume grows and fans lose confidence before event day.

Where we'd start

What we noticed

Reviews and Reddit posts show users get stuck when seller, organizer app, and StubHub status disagree. Map the ticket handoff from listing to transfer to replacement first.

Useful next step

Add event-state tests around mobile transfer, seller proof, and replacement choice. Then test a quiet checkout path before adding more purchase prompts.

What we'd do

Use a dedicated squad under StubHub's product owner for a six-week pilot, with weekly demos and one trust flow as the focus.

Team shape

  • 1 tech lead
  • 3 backend engineers
  • 1 QA automation engineer
  • 1 DevOps/data engineer

Relevant stack: Java, Node.js, Python, React, REST APIs, cloud DevOps, and test automation.

1
Trace ticket state

Trace every ticket state across listing, payment, transfer, replacement, refund, and seller payout paths in one testable service map.

2
Harden backend contracts

Add contract tests for Java or Kotlin services that publish transfer, payout, and replacement events through Kafka-like streams.

3
Protect mobile releases

Build mobile regression packs for buy, sell, upload, login, and app-region flows before each store release.

4
Speed key pages

Tune the mobile web path by cutting blocking scripts and measuring main event pages under real network limits.

5
Demo every week

Ship weekly demos with StubHub product owners, using support cases and review themes as the backlog filter.
