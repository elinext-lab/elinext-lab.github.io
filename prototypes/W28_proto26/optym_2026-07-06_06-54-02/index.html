For Rajesh Kumar

Keep LoadOps growth steady while backend hiring catches up

LoadOps links driver mobile work, dispatcher web work, GPS, documents, and payments. That makes backend quality a product issue, not only an engineering issue.

We saw Optym hiring for a Senior Software Engineer - Backend with Node.js. That points to real delivery pressure around APIs, async work, and release support.

Our read

The backend role suggests Optym is scaling product work behind LoadOps and newer freight tools. LoadOps already connects mobile drivers, the web app, GPS, documents, and payments. That is useful, but it also raises regression risk when releases move fast. Public reviews mention login trouble, document upload issues, and buggy changes after updates. If those paths stay fragile, dispatch teams lose trust and support load rises.

Where we'd start

Auth flow

One Android review says reset did not fix login. Trace mobile auth, web session, and API errors together, then add contract tests for reset states.

Document flow

A driver linked a bill-of-lading glitch to delayed pay. Make document capture, upload retry, and dispatcher visibility a release-gate smoke test.

What we'd do

A six-week backend reliability pilot

Use a dedicated squad under Optym's product and DevOps leads. The goal is to clear backend work while hardening the flows users already feel.
