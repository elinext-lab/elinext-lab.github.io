What if Sword stopped coaching with low confidence?

Recent Sword reviews are not just “bug complaints.” They show a more dangerous pattern: motion tracking gets shaky, device state breaks, therapist messaging fails, and the system keeps nudging members like the session is healthy. For a Head of Engineering in Vision AI, that is a platform-trust problem.

This prototype reframes the fix as a confidence-aware session gate: verify camera and wearable readiness before feedback starts, switch into a safe fallback when the model is uncertain, and suppress reminder escalation until the session is actually usable.

Ops / clinician surface

The value is not only in the phone. Platform teams need a small, audit-ready control plane that explains why feedback was gated and how the member was safely rerouted.

Confidence Gate rules

  • Session-level thresholding Don’t show corrective feedback below approved motion-confidence bands.
  • Device-aware fallbacks Swap from full mixed-mode coaching into camera-only or clinician-reviewed mode.
  • Outreach suppression Pause “nudge” automation while unresolved technical blockers remain active.

Expected impact

Support tickets tagged “tracker / sync / PT message”-32%
Bad-session completion attempts-41%
“AI was wrong” trust complaints-27%

Audit trail that respects the environment

  • HIPAA-aware event pack Store device / confidence / contact-state metadata with PHI minimization.
  • Telehealth / consent state Keep member consent context attached to the session resolution path.
  • SOC 2 / HITRUST evidence Make policy decisions, overrides, and handoffs exportable for audit review.
  • FDA-listed device context Never silently downgrade device-backed care into an untracked black box.
