Software reliability brief

Make the app feel as strong as the charger

myenergi now sells a full home energy ecosystem. The app must make zappi GLO, Gridpay, libbi, and installer setup feel simple every day.

Why we're here: we saw myenergi hiring senior software engineers for cloud services and React Native apps. That points to a live delivery push around device control, BLE setup, and user-facing app reliability.

Primary product: myenergi app Control layer for zappi, eddi, libbi, hub, and zappi GLO.

Our read

The hiring signal says myenergi is adding software capacity for cloud and mobile work. That fits the product moment: zappi GLO is screen-free, so the app is now the main control point. Gridpay also adds new tariff and reward logic inside the same flow. Recent Android reviews mention offline devices, slow mode changes, and schedules that do not hold. If those flows stay fragile, support load rises and users doubt the hardware.

Where we'd start

What we noticed

Users report zappi schedules disappearing or missing charge windows. Start with a regression pack for schedule create, edit, persist, Boost Now, Gridpay, and ECO+ handoff.

Useful next step

Several reviews say app energy flows differ from inverter or battery data. Add a state trace that compares device, cloud, and app values after each command.

What we'd do

A six-week reliability pilot

Use a dedicated squad under your Product Owner. The goal is not a rebuild. It is to harden the flows users already rely on.

1 mobile tech lead, 2 senior React Native engineers, 1 cloud engineer, 1 QA automation engineer, and fractional UX support.
