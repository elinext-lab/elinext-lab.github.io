Implementation Shape

Three practical moves, not a re-platform.

This is narrow enough to ship incrementally and broad enough to improve Edera now and future Kranus therapy apps after that.

1

Trusted Device State

Persist a clear device-trust state with biometric-first resume, plus better messaging when the session is still trusted.

  • device posture check
  • resume token lifecycle
  • reason-coded step-up auth
2

Recovery Without Dead Ends

Replace opaque lockouts with self-serve recovery and a transparent security center that explains what happened.

  • device handoff flow
  • in-app secure unlock path
  • office-hours support as fallback, not first resort
3

Editable Therapy Entries

Let users fix fresh mistakes with guardrails instead of forcing support or living with bad data in the progress layer.

  • short edit window
  • audit log preserved
  • better history visibility
